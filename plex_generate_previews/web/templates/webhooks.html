Configuration
Imports are queued and processed in batches. A batch runs only after this many seconds pass with no new imports from that source — so if a file arrives just before the batch would run, it goes into the queue and the timer resets; that file is processed in the next batch, after the delay has passed again. The delay gives Plex time to add the new file to its library; processing too soon can fail because Plex may not have indexed it yet. Webhook runs only process items from the libraries selected on the Settings page.
When an item from a webhook cannot be found in Plex (e.g. Plex has not indexed it yet), how many times to retry. Each retry runs as a separate job after the configured delay. Set to 0 to disable retries.
Initial delay before the first retry. Subsequent retries use exponential backoff (2x, 4x, etc., capped at the slider maximum). Gives Plex time to finish indexing.
Dedicated secret for webhook authentication. Leave empty to use the main API token.
Restart Recovery
When the server restarts, automatically re-queue any jobs that were running or waiting. Only jobs created within the configured time window are requeued; older jobs are left as-is.
How old a job can be to qualify for automatic requeue after restart (5 min – 24 hours).
Setup Instructions

  1. Go to Settings → Connect → + → Webhook
  2. Set Name: Plex Previews
  3. Set URL: copy the Radarr Webhook URL above
  4. Under Events, enable On Import and On Upgrade
  5. Authentication (use one):
    • Username/Password: Leave Username blank and put your API token or webhook secret in Password.
    • Headers: If your webhook form has a Headers section, add X-Auth-Token = your API token or webhook secret.
  6. Click Test then Save

Use the custom endpoint to trigger preview generation from any tool that can send HTTP requests. The expected JSON payload is:

{% raw %}{
  "file_path": "/media/movies/Movie (2024)/Movie.mkv"
}{% endraw %}

Or for multiple files:

{% raw %}{
  "file_paths": ["/media/tv/Show/S01E01.mkv", "/media/tv/Show/S01E02.mkv"],
  "title": "Optional display label"
}{% endraw %}

Send {% raw %}{"eventType": "Test"}{% endraw %} to verify connectivity without processing.

Tdarr Setup
  1. Open your Tdarr Flow and add a Send Web Request plugin after your transcode step
  2. Set Method: POST
  3. Set Request URL: copy the Custom Webhook URL above
  4. Set Request Headers: 
    {% raw %}{
  "Content-Type": "application/json",
  "X-Auth-Token": "YOUR_TOKEN"
}{% endraw %}
  5. Set Request Body: 
    {% raw %}{"file_path": "{{{args.inputFileObj._id}}}"}{% endraw %}
  6. Save the flow

The {% raw %}{{{args.inputFileObj._id}}}{% endraw %} template is replaced by Tdarr with the full file path of the transcoded file at runtime.
