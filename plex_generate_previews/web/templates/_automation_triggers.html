Plex Direct Webhook Instant Plex Pass required
Checking…

Plex pings this app the moment it adds a new library item, so previews start generating right away. This is the fastest option — but it only works if you have Plex Pass.

Prerequisites
  • Plex Pass active on your server-owner account.
  • Mobile Push Notifications enabled in Plex Web → Settings → General (toggle Enable mobile push notifications).
    This is a Plex quirk: the library.new webhook is delivered through the same code path as mobile push notifications, so with push notifications off, library events never fire. You don't have to actually use mobile push on a phone — they just need to be turned on.
Your Plex account doesn't appear to have an active Plex Pass subscription. Plex's webhook feature is Plex-Pass-only — use the Recently Added Scanner below instead.
Setup
  1. Confirm the URL Plex will POST to. The default is the URL you're using right now, which works for most same-host setups. Override it if Plex is on a different machine or behind a different network/proxy.
    Must be reachable from your Plex Media Server, not just your browser.
  2. Click Register with Plex below. Your webhook secret (or API token) is automatically embedded in the registered URL so Plex can authenticate — the webhook UI in Plex doesn't support custom headers, so this is the only option. If you change the webhook secret later, saving settings automatically re-registers so Plex picks up the new value.
How auth works (advanced)

When you click Register, this app calls the plex.tv account API and stores a URL like https://your-host/api/webhooks/plex?token=SECRET. Plex Media Server POSTs to that URL on server events; the endpoint filters for library.new, validates the query-string token against your webhook secret, and feeds the file paths into the same job pipeline as Sonarr/Radarr.

If you'd rather register manually, you can paste the base URL into Plex Web → Settings → Webhooks yourself — but you'll still need to append ?token=<your-secret> or Plex POSTs will 401.

*arr apps Per-import trigger

Sonarr, Radarr, and Sportarr POST to this app as soon as they finish importing or upgrading a file. Copy the webhook URL from the relevant tab below into the app's settings, add your auth token, and you're done.

Pick a server to scope these webhook URLs to it (adds ?server_id=<id>). Use this when the same library exists on more than one server (e.g. both Plex and Emby have Movies) and you want this webhook to publish previews to one specific server only.
Setup in Sonarr
  1. Go to Settings → Connect → + → Webhook
  2. Set Name: Plex Previews
  3. Set URL: copy the URL above
  4. Under Events, enable On File Import and On File Upgrade
  5. Authentication:
    • Username/Password: Leave Username blank and put your webhook secret in Password.
    • Headers (if your form has this field): Add X-Auth-Token = your webhook secret.
  6. Click Test, then Save

Custom Webhook Any HTTP client

Use this endpoint to trigger preview generation from any tool that can send an HTTP POST — Tdarr, shell scripts, Bash hooks, anything. Send a file path and the app queues it through the same pipeline as Sonarr/Radarr.

Payload — single file
{% raw %}{
  "file_path": "/media/movies/Movie (2024)/Movie.mkv"
}{% endraw %}
Payload — multiple files
{% raw %}{
  "file_paths": ["/media/tv/Show/S01E01.mkv", "/media/tv/Show/S01E02.mkv"],
  "title": "Optional display label"
}{% endraw %}

Send {% raw %}{"eventType": "Test"}{% endraw %} to verify connectivity without triggering processing.

Tdarr setup
  1. Open your Tdarr Flow and add a Send Web Request plugin after your transcode step
  2. Set Method: POST
  3. Set Request URL: copy the Custom Webhook URL above
  4. Set Request Headers: 
    {% raw %}{
  "Content-Type": "application/json",
  "X-Auth-Token": "YOUR_TOKEN"
}{% endraw %}
  5. Set Request Body: 
    {% raw %}{"file_path": "{{{args.inputFileObj._id}}}"}{% endraw %}
  6. Save the flow

The {% raw %}{{{args.inputFileObj._id}}}{% endraw %} template is replaced by Tdarr at runtime with the full file path of the transcoded file.

curl example 
{% raw %}curl -X POST "http://your-server:8080/api/webhooks/custom" \
  -H "Content-Type: application/json" \
  -H "X-Auth-Token: YOUR_TOKEN" \
  -d '{"file_path": "/media/movies/Movie.mkv"}'{% endraw %}
Settings
Master switch for all incoming webhooks (Plex Direct, Sonarr, Radarr, Custom). Disable to pause without touching any individual settings.
Imports are queued and processed in batches. A batch runs only after this many seconds pass with no new imports from that source — if a file arrives just before the batch would run, it joins the queue and the timer resets. The delay gives Plex time to finish indexing the new file before we try to process it.
When an item from a webhook can't be found in Plex (e.g. Plex hasn't indexed it yet), how many times to retry. Each retry runs as a separate job after the retry delay. Set to 0 to disable retries.
Initial delay before the first retry. Subsequent retries use exponential backoff (2×, 4×, etc., capped at the slider maximum).
Dedicated secret for webhook authentication (Sonarr/Radarr/Custom, and also embedded in the Plex Direct webhook URL). Leave empty to use the main API token instead. Rotating this value automatically re-registers the Plex Direct webhook.
Restart Recovery
When the server restarts, automatically re-queue any jobs that were running or waiting. Only jobs created within the configured time window are requeued; older jobs are left as-is.
How old a job can be to qualify for automatic requeue after restart (5 minutes – 24 hours).
