Backups

Each save to a config file (settings, schedules, webhook history) leaves a single rolling .bak next to it. If the live file ever gets clobbered — bad release, edit-by-mistake, truncation by an old container — you can restore from the most recent backup here.

Job history lives in jobs.db (SQLite) and isn't backed up via this mechanism; rolling that back is a container-revert plus optional volume-snapshot restore.
