Overview

Webhooks generate previews automatically when new media lands in Plex, Emby, or Jellyfin. Pick the trigger source below that matches how your media gets in.

You add media manually to Plex (drag & drop, Plex import, etc.)
Plex-only — Plex is the only server vendor that exposes a server-side webhook we can subscribe to.
→
Servers → Edit Plex → Webhook & Scanner Plex Direct webhook + Recently Added Scanner
You add media manually to Emby
Emby has built-in server-side webhooks (Premiere). Dashboard → Webhooks → Add Webhook, point it at https://<your-host>/api/webhooks/incoming?token=<webhook secret> and tick the Library → New Media Added event.
→
Emby Dashboard → Webhooks Posts to the same /incoming URL — vendor auto-detected.
You add media manually to Jellyfin
Install the official Webhook plugin (Dashboard → Plugins → Catalog). Add a Generic destination → URL https://<your-host>/api/webhooks/incoming?token=<webhook secret>, subscribe to the Item Added notification.
→
Jellyfin Webhook plugin → Generic destination Same /incoming URL as Plex / Emby.
Note about file upgrades: The server-side webhook options (Plex Direct + Recently Added Scanner; Emby Webhooks; Jellyfin's Webhook plugin) only fire on brand-new library items. When Sonarr or Radarr upgrades an existing file in place, the media server keeps the same library item and the new file isn't seen as "added", so the server-side webhooks don't fire. Use the Sonarr/Radarr "On Upgrade" webhook event for those replacements — it works regardless of which media server you use.
*arr apps Per-import trigger

Sonarr, Radarr, and Sportarr POST to this app as soon as they finish importing or upgrading a file. Copy the webhook URL from the relevant tab below into the app's settings, add your auth token, and you're done.

Pick a server to scope these webhook URLs to it (adds ?server_id=<id>). Use this when the same library exists on more than one server (e.g. both Plex and Emby have Movies) and you want this webhook to publish previews to one specific server only.
Setup in Sonarr
  1. Go to Settings → Connect → + → Webhook
  2. Set Name: Plex Previews
  3. Set URL: copy the URL above
  4. Under Events, enable On File Import and On File Upgrade
  5. Authentication:
    • Username/Password: Leave Username blank and put your webhook secret in Password.
    • Headers (if your form has this field): Add X-Auth-Token = your webhook secret.
  6. Click Test, then Save

Custom Webhook Any HTTP client

Use this endpoint to trigger preview generation from any tool that can send an HTTP POST — Tdarr, FileFlows, shell scripts, Bash hooks, anything. Send a file path and the app queues it through the same pipeline as Sonarr/Radarr.

To pin this URL to one configured server, use the Publish to which server? picker in the arr apps section above — it appends ?server_id=<id> to every URL on this tab.
Payload — single file
{% raw %}{
  "file_path": "/media/movies/Movie (2024)/Movie.mkv"
}{% endraw %}
Payload — multiple files
{% raw %}{
  "file_paths": ["/media/tv/Show/S01E01.mkv", "/media/tv/Show/S01E02.mkv"],
  "title": "Optional display label"
}{% endraw %}

Send {% raw %}{"eventType": "Test"}{% endraw %} to verify connectivity without triggering processing.

Tdarr setup
  1. Open your Tdarr Flow and add a Send Web Request plugin after your transcode step
  2. Set Method: POST
  3. Set Request URL: copy the Custom Webhook URL above
  4. Set Request Headers: 
    {% raw %}{
  "Content-Type": "application/json",
  "X-Auth-Token": "YOUR_TOKEN"
}{% endraw %}
  5. Set Request Body: 
    {% raw %}{"file_path": "{{{args.inputFileObj._id}}}"}{% endraw %}
  6. Save the flow

The {% raw %}{{{args.inputFileObj._id}}}{% endraw %} template is replaced by Tdarr at runtime with the full file path of the transcoded file.

FileFlows setup

In FileFlows, install the Web plugin (Plugins page → search "Web") so the Web Request node becomes available, then add two nodes to the end of your flow:

  1. Add a Set Variable node just before the Web Request node — FileFlows doesn't expose a built-in "final output path" variable, so build it from the folder + final filename + extension:
    • Variable: output_file
    • Value: {% raw %}{folder.Orig.FullName}/{file.NameNoExtension}{ext}{% endraw %}
  2. Add a Web Request node after the Set Variable node:
    • Method: POST
    • URL: copy the Custom Webhook URL above
    • Content Type: JSON
    • Headers: key X-Auth-Token, value YOUR_TOKEN
    • Body: 
      {% raw %}{"file_path": "{output_file}"}{% endraw %}
  3. Save the flow

The {% raw %}{output_file}{% endraw %} placeholder is substituted by FileFlows at runtime with the absolute path of the file as it exists at the end of the flow (after any rename / re-extension steps).

curl example 
{% raw %}curl -X POST "http://your-server:8080/api/webhooks/custom" \
  -H "Content-Type: application/json" \
  -H "X-Auth-Token: YOUR_TOKEN" \
  -d '{"file_path": "/media/movies/Movie.mkv"}'{% endraw %}
Settings
Master switch for all incoming webhooks. Turn off to pause without touching any individual settings.
Imports are queued and processed in batches once this much time passes without new imports.
Dedicated secret for webhook authentication (Sonarr/Radarr/Custom, and also embedded in the Plex Direct webhook URL configured in Servers → your Plex card → Webhook & Scanner). Leave empty to use the main API token instead. Rotating this value automatically re-registers any Plex Direct webhook the next time you save it from the Servers page.

Retry policy, restart recovery, and queue concurrency have moved to Settings → Processing → Job Execution. Those controls apply to every job type, not just webhooks.

Inbound webhook events and their outcomes are tracked under Tools. View webhook activity
