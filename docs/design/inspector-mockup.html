Per-server state

Eight distinct UX situations the Inspector must handle gracefully. Each card mirrors what a user will see in their Servers tab.

Plex — Home Trigger native
Plex Pass + native detection on

Server auto-detects markers. We read them, surface anything Plex missed. Trigger re-analysis with one click; we never write to the DB.

Plex — Family Direct DB write
No Pass — direct SQLite write

We write to taggings + media_parts.extra_data using the MarkerEditor pattern. Provenance/restore loop survives re-analysis. Big warning before first write.

Emby — Office Plugin write
Premiere + Segment_Reporting plugin

Native intros (10-min ceiling) + our credits/recap via POST /emby/segment_reporting/update_segment. Both sources merged in the timeline.

Emby — Spare Read only
Premiere but plugin missing

We can read native chapters + markers but can't write. Setup Health flags this with a one-click install link to sydlexius/Segment_Reporting.

Jellyfin — Main Plugin write
MediaPreviewBridge v10.11.1.0+

Full integration. Sidecar JSON written next to media; POST /MediaPreviewBridge/Markers/{id} + scheduled scan keep MediaSegments in sync.

Jellyfin — Lab Update needed
MediaPreviewBridge needs upgrade

Existing trickplay plugin is too old (10.11.0.2) for marker writes. One-click upgrade; same manifest URL, no re-install.

Jellyfin — Friend Read only
No plugin installed

Read-only mode. Inspector still shows markers Jellyfin already has + everything we detected. Install instructions include a paste-ready manifest URL.

Server unreachable Offline
Connection failing

Last seen 6m ago. Detection still runs; markers queue in our sidecar and publish when the server is back.

Integration gaps that need explicit design before Phase B/C:
  • Frame cache reuse — can Tier 3 blackdetect use BIF's already-extracted frames?
  • GPU pool sharing — detection has max_concurrent_detections; how does it negotiate with BIF workers?
  • SocketIO progress events — BIF emits per-frame; what does detection emit?
  • Setup wizard — markers not yet in setup.html onboarding flow
  • Scheduler — no marker-backfill task in APScheduler yet

Edge cases

Be honest when there's no signal. Reality TV and mini-series with unique cold opens get a REJECT verdict — not a wrong marker.

REJECT
Reality TV

Survivor / Real Housewives / MAFS — no recurring intro audio. We don't try. Inspector shows "no markers detected" with a category-specific explanation.

PARTIAL
Mini-series

Chernobyl / Watchmen — chromaprint fails (unique cold opens) but TheIntroDB has data 60% of the time. Surface Tier 1 hit with confidence.

LOCKED
Manual edit

You nudged S01E03's intro end. Re-detection runs anyway, but produces a parallel candidate. Inspector shows the diff; you choose.

BRIDGE
Plex → Jellyfin

Plex Pass user wants Jellyfin to skip too. Read Plex's markers via plexapi, push to Jellyfin via our plugin. One click. No re-detection.

RATE LIMIT
TheIntroDB 429

500 lookups/day burned. We don't block — fall back to Tier 2/3 chromaprint immediately. Inspector shows when quota resets.

PLUGIN MISSING
Jellyfin no MediaPreviewBridge

Read-only mode. Inspector still shows everything we detected; "Apply" replaced by "Install plugin" with paste-ready manifest URL.
